## гҖҢ What is MagicalGirl SH? гҖҚ

**MagicalGirl SH** is a high-fidelity **AI-driven Web JRPG** prototype. Unlike traditional JRPGs with pre-scripted dialogue trees, every line of narrative, every NPC reaction, and every battle outcome is dynamically generated by a large language model вҖ” producing a truly emergent storytelling experience that adapts to your choices in real time.

The game fuses:

- **AI Dungeon Master** via Vercel AI SDK вҖ” the LLM orchestrates narrative, manages game state, and executes tool calls
- **Shin Megami Tensei-inspired Press Turn combat** with elemental affinity bitmasking, status effects, and passive skill trees
- **Full player character creation** вҖ” name, gender, appearance (body type / hair / eyes), and background story
- **SQLite WASM persistence** running in a Web Worker for save/load with zero backend
- **PixiJS 8 animated backgrounds** with WebGL rendering and performance-friendly fallbacks
- **Multi-theme UI** вҖ” E-girl / E-boy, Kidcore, and Pastel Brutalism

It's a **single-page JRPG engine** where the AI is your GM, your combat arbiter, and your co-writer.

### рҹҺӯ AI-Powered Narrative

The orchestrator layer (`src/orchestrator/`) constructs structured prompts from game state, character data, world lore, and conversation history, then feeds them through the LLM. The model can invoke tool calls вҖ” `update_variables`, `trigger_battle`, `read_skill` вҖ” allowing it to actively manipulate the game world rather than merely describe it.

### вҡ”пёҸ Press Turn Combat System

A faithful adaptation of the **Press Turn** battle engine from *Shin Megami Tensei*, featuring:

| Mechanic                 | Description                                                               |
| ------------------------ | ------------------------------------------------------------------------- |
| **Elemental Affinity**   | Bitmask-based; `WEAK` / `NEUTRAL` / `RESIST` / `NULL` / `DRAIN` / `REPEL` |
| **Turn Economy**         | Hit a weakness вҶ’ gain extra press turn; get nulled вҶ’ lose turns           |
| **Status Effects**       | Poison, Paralysis, Charm, Fear, and more вҖ” each with unique turn logic    |
| **Passive Skills**       | Passive effects that trigger under specific battle conditions             |
| **Formula-Based Damage** | Stats-driven damage calculation with skill power multipliers              |

### рҹҺЁ Character Creator

A multi-step wizard lets players define every detail of their persona:

- **Name & Gender** вҖ” with AI-generated fallback defaults
- **Appearance** вҖ” body type, hair style, eye color, all visually previewed
- **Background** вҖ” family history, past experience, shaping the AI's narrative tone

### рҹ’ҫ SQLite WASM Persistence

Game state is persisted via **sqlite-wasm** running in a dedicated Web Worker:

- Full save/load with multiple save slots
- Chat history, world variables, inventory, skill data вҖ” all in one embedded DB
- Graceful fallback to in-memory stores when WASM is unavailable

### рҹҢҗ Lorebook / World Info

A content registry backed by JSONL data files that injects contextual lore into AI prompts based on game state conditions вҖ” ensuring the AI remembers the world's rules, factions, and characters.

## гҖҢ Getting Started гҖҚ

```bash
# Clone the repo
git clone https://github.com/henryd2341/magicalgirl-sh.git
cd magicalgirl-sh

# Install dependencies (pnpm recommended)
pnpm install

# Start dev server
pnpx vite

# Run tests
pnpm test

# Run E2E tests
pnpm test:e2e

# Lint
pnpm lint
```

> **Prerequisites:** Node.js вүҘ 18, pnpm вүҘ 8

## гҖҢ Architecture at a Glance гҖҚ

```
src/
в”ңв”Җв”Җ orchestrator/    # AI prompt builder, harness tools, state management
в”ңв”Җв”Җ engine/          # Battle engine (Press Turn, damage formulas, status effects)
в”ңв”Җв”Җ components/      # Vue components (chat, battle UI, character creator, HUD)
в”ңв”Җв”Җ stores/          # Pinia stores (game state, settings, world info)
в”ңв”Җв”Җ workers/         # Web Worker for SQLite WASM
в”ңв”Җв”Җ data/            # JSONL world info / lorebook data
в””в”Җв”Җ themes/          # SCSS theme definitions
```
